Why PicoContainer Script?

The main scope of PicoContainer Script to provide a scripting layer on top of PicoContainer, adding the following features:

  • Classloader (classpath) management,including programmable permissions
  • Class name based composition (via reflection)
  • Meta-data and script language support:
    • XML
    • Beanshell
    • Groovy
    • JRuby
    • Jython (Python)
    • Rhino (Javascript)

It is important to stress that there is no "official" script or meta-data language. We support multiple script languages to give the use the maximum flexibility in the choice that suits the task at hand. XML is of course the most widely used and mature meta-data markup, but lacks the programming features of some scripting languages, such as Groovy or JRuby with its builder syntax.
