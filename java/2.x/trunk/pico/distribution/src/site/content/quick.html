Quick Introduction to Use

There is an assumption here that you know what Dependency Injection and Inversion of Control are.

Constructor Dependency Injection (CDI)

With PicoContainer you add components by type or instance, then you get components out. Pico works out the dependancies, or tells you via exception that it cannot. 

pico = new DefaultPicoContainer();
pico.addComponent(ComponentOneImpl.class) // by type
pico.addComponent(ComponentTwoImpl.class) // by type
pico.addComponent(new ComponentThreeImpl()) // by instance
pico.addComponent(ComponentFourImpl.class) // by type

ComponentFourImpl four = pico.getComponent(ComponentFourImpl.class);

It is as likely, you will be honoring the facade pattern for your component design and having interface/impl separations.

pico = new DefaultPicoContainer();
pico.addComponent(ComponentOne.class, ComponentOneImpl.class) // by type
pico.addComponent(ComponentTwo.class, ComponentTwoImpl.class) // by type
pico.addComponent(ComponentThree.class, new ComponentThreeImpl()) // by instance
pico.addComponent(ComponentFour.class, ComponentFourImpl.class) // by type

ComponentFour four = pico.getComponent(ComponentFour.class);

You can use a fluent style if you want:

ComponentFour four = new DefaultPicoContainer()
    .addComponent(ComponentOne.class, ComponentOneImpl.class)
    .addComponent(ComponentTwo.class, ComponentTwoImpl.class)
    .addComponent(ComponentThree.class, new ComponentThreeImpl())
    .addComponent(ComponentFour.class, ComponentFourImpl.class)
    .getComponent(ComponentFour.class);
