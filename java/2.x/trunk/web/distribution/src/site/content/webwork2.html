To configure PicoContainer to handle the dependency injection of WebWork2, refer to the fragment of web.xml below. For a start there is a slightly different listener to the one defined in the main page. Also, instead of using WebWork2's servlet, use ours. Everything else from the standard WebWork2 setup is appropriate.


  org.picocontainer.web.webwork2.WebWork2PicoServletContainerListener



  WebWork2
  org.picocontainer.web.webwork2.PicoWebWork2ServletDispatcher



  WebWork2
  *.action
