What is a component ?

A component is a class that, for PicoContainer at least, is characterized in the following ways :

Doesn't have to implement or extend anything
Declares one or more constructors where the arguments are the component's dependencies.
Reusable in many different deployments

1. Constructors

One or more constructor that many have other components as parameters.

The constructor is not mandadatory. PicoContainer will use the constructor with the largest number of satisfiable arguments. Currently, primitive (int, boolean ets.) types cannot be used, but their class counterparts can.

2. Reusable in many different deployments

The ability to be instantiated in a variety of deployment scenarios (root, servlet, applet ClassLoaders etc)

Probable (but not mandatory) interface/implementation separation.
