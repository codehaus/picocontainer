Overview

A component is a class that, for PicoContainer at least, is characterized in the following ways :
  1. Does not have to implement or extend anything.
  2. Declares one or more constructors where the arguments are the component's dependencies and configuration.
  3. Reusable in many different deployments.

1. Transparent



2. Constructors

One or more constructor that many have other components and configuration as parameters.

The constructor is not mandadatory. PicoContainer will use the constructor with the largest number of satisfiable arguments.

3. Reusable in many different deployments

The ability to be instantiated in a variety of deployment scenarios (root, servlet, applet ClassLoaders etc)

Probable (but not mandatory) interface/implementation separation.
