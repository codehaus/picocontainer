DefaultLifecyclePicoAdapter (Java)

See lifecycle

This class adapts a PicoContainer instance to have start() and stop() capabilty. If start() is called, and any of the contained components implementes "Startable" then start() is called on that component.  DefaultLifecyclePicoAdapter is one implementation of LifecyclePicoAdapter. If any other concept of start/stop is required, then this class deomostrates how to do it.

DefaultLifecyclePicoContainer (Java)

See lifecycle

This true PicoContainer takes a LifecyclePicoAdapter and a MutablePicoContainer in its constructor. It aplies one to the other and represents itself as both to the class that instantiated it.

ImplementationHidingComponentAdapterFactory & ImplementationHidingComponentAdapter (Java)

It is usual for containers hosting unknown components (think Applet) to want to protect itself from hacking. One way element of the protect is to hide the implementations of an interface. The ImplementationHidingComponentAdapter does this by using reflection's Dynamic Proxy facility.

ThrowawayPicoInstantiator (Java)

If you have single component that you want to instantiate with unknown component dependencies, and the container itself is not important, then ThrowawayPicoInstantiator is for you:

Class someFooImplementationClass = // from somewhere
Foo foo = (Foo) new ThrowawayPicoInstantiator(parentContainer, clazz).getInstance();
