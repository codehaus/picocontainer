Introduction

PicoContainer is very simple container for very simple components.  It honors the Constructor Dependency Injector form of the Inversion of control pattern (IoC).  Components are typically going to live inside the same JVM. The idea is that this scales from embedded containers for simple beans to enterprise and distributed applicagtions.

Simple Example

In skeletal form, here is the simplest possible PicoContainer compatible component:
   public class Shop {
       public Shop(StockManager stockManager) {
       }
   }
Imagine there were a component that could satisfy that like so:
   public class StockManager {
   }

Use without a container

We love the Pico design because we see the components being used without a container:
   new Shop(new StockManager());
This is particularly good for unit testing. In-container testing being one of the most avoided forms of unit testing.  This is where other container designs and specifications fail.

Use with a container

With a PicoContainer:
   PicoContainer pc = SomePicoContainer();
   pc.registerComponent(StockManager.class)
   pc.registerComponent(Shop.class)
   pc.start();
It does not matter how things are laced together. It does not matter how tightly bound one PicoContainer is to an individual purpose.  The important thing is actually the components and the numerous and imaginative deployments for them.

Features

PicoContainer supports the following set of compelling features:
  • Dependent components are laced together through constructor parameters
  • Dependent components can also (optionally) be passed as JavaBean properties
  • Configuration of components (based of the above)
  • Default lifecycle support built in (start, stop, dispose), can also be extended with additional states
  • Very extensible design (through for example the ComponentAdapter), enabling virtually any form of extensions to the core

Sister projects

PicoExtras

PicoExtras extend PicoContainer with the following features, things like:
  • AOP (through integration with Nanning)
  • XML configuration
  • Hot-swapping
  • ...and much more

NanoContainer

NanoContainer is a container that incorporates most of the PicoExtras extensions into one single runnable container, almost like an application server.

Ports

.NET

Pico.NET is the port of PicoContainer to .NET. It follows the Java design closely. Currently a work in progress by Maarten Grootendorst.

Ruby

Rico is the port of PicoContainer to Ruby by Dan North. Ruby is a dynamically typed lanuage, so the Pico design does not map exactly.

PHP

A port to PHP 5 is currently in the progress of being written by Sebastian Bergmann.
